# 06 — Dashboard Concept

An interactive concept mockup built during the brainstorm to make the design tangible.

- **File:** [`dashboard-mockup.html`](dashboard-mockup.html) (this folder)
- **Live:** https://claude.ai/code/artifact/c384d238-6823-4e85-9667-671dbb57fe25
- **Data:** clocks and open/closed logic are **real**; prices, movers, news, and AI text are **sample**.

> **✅ Build status (v1):** The mockup has been implemented as the real app. The exact CSS/design
> language was ported verbatim into `frontend/css/styles.css`, and the single-file `MK` / `MOVERS` /
> `NEWS` / `ADVICE` / `WL` objects are now the modular `frontend/js/ui/*` render modules fed by the
> backend API (crypto prices genuinely live). Run it via the root [`README.md`](../../README.md).

## Design language
- **Name/brand:** "Stocks Eye" — an eye that watches markets (👁).
- **Palette:** deep slate-navy ground, raised surfaces, **gold accent** (the one interactive/brand
  hue, and the marker for the AI features). Semantic **gain green / loss red**, always paired with
  ▲/▼ + sign so meaning never depends on color alone.
- **Type:** system UI for labels, a **monospace** for all numbers (financial-terminal feel + clean
  column alignment via tabular figures).
- **Layout:** AI Morning Brief on top → live market-status tiles → selected-market detail (chart +
  AI Take + movers + news) beside a USD-normalized watchlist → data-routing footer.

## What's in it
### 1. AI Morning Brief (top, roll-up)
Timestamped card: lead paragraph, a one-liner per market with a sentiment marker, a "Today's hint,"
citation chips (Reuters, Bloomberg, Argaam, Kitco, CoinDesk) + a "+ live web search" chip, and the
"Generated by AI (Opus 4.8) · not investment advice" line. See [03 — AI Advice](03-ai-advice.md).

### 2. Live market-status tiles (the hero)
Seven tiles (USA, KSA, UAE, Egypt, China, Gold, Crypto). Each shows flag, name, **local time ticking**,
an **Open/Closed pill computed live** against the market's real hours *and trading week*, the index +
% change, a sparkline, and a **"Watch" row** of suggested symbols/coins. Grid auto-fits and wraps.
- Tadawul & EGX show as **Sun–Thu** markets; China has a midday break; **Gold ~24h Mon–Fri**;
  **Crypto shows "24/7" and never closes.**

### 3. Selected-market detail (click any tile)
- Area chart of the index/headline.
- **AI Take · [Market]** — per-market advice with its own sentiment tag + citations (separated per
  market, per the design decision).
- **Top movers** for that market (for crypto: the top 5 coins).
- **Latest · news & sentiment** — 3 headlines with ▲/▼/● markers + source + "not advice" disclaimer.

### 4. Watchlist (USD-normalized)
Cross-market symbols with native currency (﷼, £, ¥, د.إ, $) **and** the USD conversion beside it —
demonstrating the currency-normalization requirement.

### 5. Data-routing footer
The source-per-market plan from [01](01-data-sources.md), with **UAE flagged in gold as the open gap**,
and CoinGecko/metals for crypto/gold.

## From mockup → real build
The mockup is a **single HTML file** with `MK` / `MOVERS` / `NEWS` / `ADVICE` / `WL` objects. In the
real app these split into the **modular per-market / per-service files** described in
[04 — Architecture](04-architecture.md): `/markets/*.js` configs, `/services/**` adapters, `/core/*`
(normalizer, router, cache, market-hours, currency), and `/ui/*` modules.
