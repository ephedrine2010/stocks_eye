# 👁 Stocks Eye

A multi-market monitoring dashboard — **USA, KSA, UAE, Egypt, China, Gold, and Crypto** — with
live open/closed status, prices, per-market news & sentiment, a **per-market AI Take**, and a daily
**AI Morning Brief**.

Built from the brainstorm in [`documentation/brainstorm/`](documentation/brainstorm/).

## Quick start

```bash
npm install
npm start
# → open http://localhost:3000
```

Runs **keyless** out of the box:
- **Crypto** — fully live via **CoinGecko** (top 5 coins, 24/7).
- **Equities + Gold** — live via **Yahoo**, with a graceful **mock fallback** (never a blank tile).
- **News** — RSS where configured, otherwise sample headlines.
- **AI Take / Morning Brief** — grounded **mock** text until you add an Anthropic key.

## Enable the real AI layer (optional)

```bash
cp .env.example .env
# put your key in .env:  ANTHROPIC_API_KEY=sk-ant-...
npm i @anthropic-ai/sdk
npm start
```

With a key, the per-market AI Take + Morning Brief are generated by **Claude (Opus 4.8)** using the
web-search tool, grounded on the market snapshot (never inventing prices). See
[`documentation/brainstorm/03-ai-advice.md`](documentation/brainstorm/03-ai-advice.md).

## Project structure (modular by design)

```
backend/
  server.js               Express app (API + static frontend)
  core/                   cache · marketHours · currency · normalizer · router · env
  markets/                one config file per market (usa, ksa, uae, egypt, china, gold, crypto)
  services/
    prices/               coingecko (live) · yahoo (live) · mock (fallback)
    news/                 rss + mock fallback
    ai/                   per-market take + roll-up brief (mock / guarded Claude)
    mock/                 sample dataset
    dashboard.js          aggregator — the only place sources are stitched together
  routes/api.js           /api/dashboard · /api/markets · /api/brief
frontend/
  index.html
  css/styles.css
  js/                     api · format · marketHours · charts
    ui/                   brief · tiles · detail · watchlist
    main.js               orchestrator
```

**The rule:** add or swap a data source → touch **one file** in `backend/services/**`.
Add a market → **one file** in `backend/markets/` + one line in `markets/index.js`.

## API

| Endpoint | Returns |
|----------|---------|
| `GET /api/dashboard` | `{ markets, watchlist, brief, as_of }` — everything the UI needs |
| `GET /api/markets` | just the markets array |
| `GET /api/brief` | just the roll-up brief |
